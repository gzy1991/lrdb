本文档提供前端框架的各种规范说明

静态框架结构

路径为/src/main/webapp/static

目录结构

static/
    app/
    core/
        home/
        login/
        ....
    global/
        config.js
        base.js
    modules/
        ztree/
        util/
        ....
    index.html
    main.js

功能划分

  • app ---- 应用级目录, 用于开发项目需求功能
  • core ---- 系统级目录, 提供系统基础功能
  • global ---- 提供require配置和全局方法
  • modules ---- 前端组件目录

系统入口

  • 'static/'目录下的'index.html'和'main.js'为静态开发模板的入口文件
  • 'static/home/'目录下的'home.jsp'为部署模式下系统入口文件

组件参数命名规范

统一通用参数的命名

名称 功能
key 自定义数据字段
multi 单选or多选
data 组件数据源
initData 初始化数据
simpleData 可接受初始化简单数据
simpleReturnData 返回简单数据
dataRefill 回填数据
